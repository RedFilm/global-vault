# GTM-kit и запуск первых продаж

## 1) Позиционирование (one-pager)

### Продукт
Voice-first база знаний для производства: превращает опыт сотрудников в структурированные кейсы и позволяет быстро находить похожие решения.

### Для кого
- Производственные площадки с повторяющимися нетиповыми инцидентами.
- Сервисные инженерные команды.

### Ценность
- Снижение времени повторной диагностики.
- Снижение потерь из-за простоев.
- Сохранение критичных знаний при смене персонала.

### Почему сейчас
- Высокая стоимость простоев и дефицит экспертов.
- Рост интереса к AI-инструментам в операционных процессах.

## 2) Demo-сценарий (15-20 минут)

### Шаг 1: Захват знания (5 минут)
- Пользователь надиктовывает инцидент.
- Система показывает автоструктуру кейса.
- Публикация в базу в 1-2 клика.

### Шаг 2: Поиск повторного решения (5 минут)
- Пользователь формулирует голосовой запрос.
- Система выдает похожие кейсы с объяснением релевантности.
- Пользователь отмечает полезность результата.

### Шаг 3: Управление качеством (5 минут)
- Эксперт переводит кейс в "Проверено".
- Руководитель видит базовые KPI по внедрению.

### Шаг 4: Бизнес-эффект (3-5 минут)
- До/после по времени диагностики.
- Пример сохраненного простоя в деньгах.

## 3) ROI-калькулятор (упрощенная модель)

### Входные данные
- Количество повторных инцидентов в месяц.
- Среднее время повторной диагностики до внедрения.
- Целевое сокращение времени (например, 20%).
- Стоимость часа простоя.

### Формула
- Сэкономленные часы в месяц =
  `incidentsPerMonth * avgDiagHours * reductionPercent`
- Экономия в месяц =
  `savedHours * downtimeCostPerHour`
- Годовая экономия =
  `monthlySavings * 12`

### Решение для клиента
- Если годовая экономия >= 3x стоимости подписки, пилот считается экономически оправданным.

## 4) Тарифная логика (для старта)
- Базовый: за площадку + лимит пользователей.
- Профессиональный: за площадку + расширенные роли и аудит.
- Enterprise: on-prem, расширенная безопасность, SLA, интеграции.

Рекомендуемый подход:
- Первые 2-3 клиента: пилот по фиксированной цене + success criteria.
- После подтверждения ROI: переход на годовой контракт.

## 5) Воронка первых продаж
- Этап 1: Lead qualification (боль, масштаб, ЛПР, бюджет).
- Этап 2: Discovery call + оценка baseline.
- Этап 3: Demo на реальном сценарии клиента.
- Этап 4: Коммерческое предложение и рамки пилота.
- Этап 5: Пилот с weekly review.
- Этап 6: Контракт и расширение.

## 6) Критерии квалификации лида (BANT+)
- Budget: есть бюджет на операционную эффективность/ИТ.
- Authority: доступ к ЛПР.
- Need: подтвержденная повторяемая проблема.
- Timeline: готовность стартовать пилот в 1-2 месяца.
- Data/IT readiness: минимальная готовность контура.

## 7) Работа с возражениями
- "Сотрудники не будут вносить кейсы":
  - Ответ: voice-first поток 60-90 секунд + регламент команды.
- "AI ошибается":
  - Ответ: экспертная валидация, прозрачные источники, статус качества.
- "Требуется высокий уровень безопасности":
  - Ответ: ролевая модель, аудит, roadmap/on-prem.
- "У нас уже есть wiki":
  - Ответ: wiki не решает быстрый поиск и структурирование operational knowledge.

## 8) Пакет коммерческих материалов
- One-pager продукта.
- Шаблон КП (цели, объем, KPI, цена, сроки).
- Шаблон pilot agreement с критериями успеха.
- Demo-скрипт для пресейла.
- ROI-калькулятор в таблице.

## 9) 30/60/90-дневный план продаж

### 0-30 дней
- Сформировать список 30 целевых компаний.
- Провести 10 discovery-встреч.
- Назначить 5 demo-сессий.

### 31-60 дней
- Запустить 2-3 пилота.
- Вести weekly review по KPI.
- Собрать 2 case studies.

### 61-90 дней
- Закрыть минимум 1 коммерческий контракт.
- Подготовить расширение в соседние подразделения.
- Уточнить ценностное предложение на основе результатов.
