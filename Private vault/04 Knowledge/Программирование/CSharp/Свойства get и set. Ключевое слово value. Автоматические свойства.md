# Свойства get и set. Ключевое слово value. Автоматические свойства.

Свойства - это один из механизмов в c#, реализующий инкапсуляцию. Свойства необходимы для описания взаимодействия с полями классов. Когда переменная private типа, но мы хотим, чтобы из другого класса с ней всё же можно было что-то сделать, пишутся методы (геттеры и сеттеры):

![[Private vault/04 Knowledge/Documents/c7b012529fe505aeabb66f67e7d6337d_MD5.png]]

Фактически использовать сеттеры и геттеры не запрещено, но в C# существует специальный механизм, который позволяет реализовывать все это, называемый свойством. Отличие их в том, что нам не надо на уровне экземпляра класса вызывать какие-то методы, а выглядеть это будет так, как будто мы работаем с обычной переменной напрямую. Для того, чтобы быстро написать свойство, достаточно ввести propfull и 2 раза нажать Tab, после чего появится готовое свойство:

![[Private vault/04 Knowledge/Documents/ade1fdb36562b468e9dccbfd9577f55f_MD5.png]]

Get и set здесь называются акцессорами. Ключевое слово value - это значение, которое мы присваиваем из вне, когда обращаемся к этому свойству:

![[Private vault/04 Knowledge/Documents/a90c36374858d62449aef5ee1e58cd8f_MD5.png]]

Благодаря свойствам можно описывать дополнительную логику поведения полей, например при записи в них:

![[Private vault/04 Knowledge/Documents/354484ffa0ab6e3d0d611194f19e5c29_MD5.png]]

Очень часто возникает ситуация, когда нам не нужна дополнительная логика поведения поля, нужно лишь получать к нему доступ, напрямую сделать поле public - это очень плохая практика, оно всегда должно быть private, для этого есть автосвойства. Prop и 2 раза Tab:
