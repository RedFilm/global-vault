# Abstract class
[[Ключевые слова и модификаторы|К разделу]] | [[Virtual Override|← Предыдущая]] | [[Interface|Следующая →]]


Абстрактный класс - базовый класс, который не предполагает создание экземпляров и может содержать в себе не реализованные члены.

Абстрактный класс описывает некую абстракцию, например «Транспорт». Некоторые сущности, которые мы хотим выразить с помощью языка программирования, могут не иметь конкретного воплощения. Например, в реальности не существует геометрической фигуры как таковой. Есть круг, прямоугольник, квадрат, но просто фигуры нет. Однако же и круг, и прямоугольник имеют что-то общее и являются фигурами. И для описания подобных сущностей, которые не имеют конкретного воплощения, предназначены абстрактные классы.

Такие классы могут иметь абстрактные члены (методы, свойства, индексаторы, события), которые не требуют реализации в данном классе, но должны быть реализованы в классах-наследниках. Здесь есть ещё одна особенность, производный класс обязан реализовать все абстрактные члены родительского, но он может этого и не делать, если сам будет абстрактным.

Реализация абстрактных членов в производных классах производится с использованием слова **«override»**.
