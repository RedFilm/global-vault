Stack и Heap (стэк потока и управляемая куча) - области оперативной памяти, но используются по-разному, для разных целей и имеют разный объём ОЗУ.

**Stack** - небольшой объём оперативной памяти, выделяющийся для каждого потока (обычно 1 Мб). Стек используется для того, чтобы выполнять логику работы программы. Работает по принципу LIFO (Last in first out - последняя поступившая задача будет выполнена в первую очередь). В нём хранятся только значимые типы. В стеке можно очень быстро разместить данные и потом также быстро освободить память от этих данных. При создании переменной значимого типа она сразу попадает в стек, а когда она пропадает из области видимости (к примеру, функция заканчивается), эта переменная автоматически удаляется из стека.

**Управляемая куча** - область оперативной памяти, имеющая больший размер (для 32-разрядного процесса около 1.5 Гб памяти, для 64-разрядного до 8 Тб). В ней хранятся только ссылочные типы. В управляемой куче нам накладнее выделять память для данных и, когда они уже не нужны, нам более проблематично освободить используемую память. В С# освобождением памяти от ненужных данных занимается сборщик мусора, В С++, например, этим занимается сам разработчик, если об этом не позаботиться, произойдёт утечка памяти.

Таблица значимых и ссылочных типов

Для значимых типов при копировании значения одной переменной в другую в стеке выделяется память под вторую переменную с таким же значением.

![[Private vault/04 Knowledge/Documents/6ab1ee1cc96c949c8998d3c68e737cce_MD5.png]]

Для ссылочных типов при копировании значения одной переменной в другую в стеке выделяется место под вторую ссылку, но сами данные (хранимые в куче) не дублируются. Т.е. при таком копировании получаем две одинаковые ссылки на одну и туже область памяти с реальными данными.
